1) client/src/lib/financial-calculations.ts
1.1 calculateMaximumWithdrawal ersetzen (Median-konsistent)

Ersetze die gesamte Funktion durch:

/** PMT-basierte maximale Entnahme – median-konsistent */
function calculateMaximumWithdrawal(
  startingCapital: number,
  months: number,
  inflationAdjusted: boolean, // ungenutzt – Entnahme ist immer nominal konstant
  portfolio: Strategy
): number {
  const { net, sigma } = getAnnualParamsForStrategy(portfolio);

  // annual sigma (p.a.)
  const sigma_annual = sigma;

  // Median-konsistente monatliche Log-Drift:
  // 12*m + 0.5*sigma_annual^2 = ln(1+net)
  const m_month = (Math.log(1 + net) - 0.5 * sigma_annual * sigma_annual) / 12;
  const r_med_month = Math.exp(m_month) - 1; // Median-Monatsrendite

  // PMT mit dieser Monatsrendite
  return startingCapital * r_med_month / (1 - Math.pow(1 + r_med_month, -months));
}


Hintergrund: Die Simulation nutzt lognormale Returns. Die median-konsistente Monatsrendite stellt sicher, dass der Medianpfad die PMT bis zum Endalter trägt.

1.2 generateWithdrawalTimeline – Drift korrekt & Entnahme konstant nominal

Suche in generateWithdrawalTimeline(...) den Block, in dem mu_log_m/sigma_m gesetzt werden, und ersetze ihn durch:

  const { net, sigma } = getAnnualParamsForStrategy(portfolio);
  const sigma_annual = sigma;
  const stepsPerYear = 12;

  // Korrektes Mapping: effektive Jahresrendite -> monatliche Log-Drift
  // 12*m + 0.5*sigma_annual^2 = ln(1+net)

Stelle sicher, dass die Entnahme je Monat konstant nominal ist (kein Inflationsfaktor):

4) Validierung (kurzer Check)

Schritt A: Rechne einen typischen Case (z. B. Startkapital 515.000 €, Entnahme 25→90).
Erwartung: Median-Linie im Entnahme-Chart fällt erst im Endjahr auf ≈ 0 €.

Schritt B: Reduziere testweise die Volatilität (z. B. σ=0), dann muss die PMT exakt dem deterministischen Ergebnis entsprechen.

Schritt C: Erhöhe σ → PMT bleibt gleich (nominal konstant), aber die Median-Kurve kann minimal von 0 abweichen (numerische Toleranz ≤ wenige Euro).

Falls punktgenaues „0 €“ gewünscht wird, kann optional ein kurzer Bisection-Solver (10–15 Iterationen) die PMT auf MedianEnd≈0 feinjustieren. Ich liefere bei Bedarf gern den Zusatz.
